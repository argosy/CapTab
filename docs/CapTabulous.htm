Platform Overview

The CapTabulous Platform is for managing and analyzing all aspects of a companys capitalization table, from the mundane maintenance of securities outstanding and current shareholdings, to the more complex ProForma modeling and Waterfall/Return Curve generation.

It is the first tool ever released that supports virtually all known vehicles used to finance venture-backed startups, from common stock, to multiple convertible notes/SAFEs, to options on a capped participating preferred with an accruing liquidation preference. It was intentionally written in Excel where it belongs, so it is user extensible and can leverage Excel's world class modeling functionality. CapTabulous is the result of a 6 year passion project of its author, enabled by his extensive background in applied mathematics, computational theory, and structured venture finance, which includes his 4 years of concentration as a Harvard undergraduate studying applied math and computer science, an MBA from Harvard Business School with focus on applied finance (with a masters thesis in collaboration with nobel laureate Fischer Black), 2 years writing fixed income arbitrage algorithms for wall street, 20 years authoring commercial network and communications platforms published by Symantec, Novell, and Citrix, and 30 years as a venture capital investor. The author is also one of the longest tenured users of Microsoft Excel, having been a beta tester of its first release in 1984, which was co-authored by his college roommate! The author also holds several fundamental patents on email and computer vision technology.

Design Philosophy: CapTabulous was born of the authors frustration with anemic web-based cap table maintenance and analytics platforms that require the user to ping pong back and forth through multiple pages to find a companys capitalization and associated terms, and that lack of support for the full spectrum of security types (and have even more limited support for convertible notes). Failure to support a singe security type, interest rate calc, or note conversion makes creating an accurate return curve impossible to produce. CapTabulous is adaptable and comprehensive enough to support virtually all equity and debt constructs, and was painstakingly written to exist with the worlds leading modeling tool (Excel), allowing for unlimited user extensibility. The goal was to present the user with all the info they could possibly need in a single view (presuming a large monitor), while providing the import and export of any data (Carta cap table imports are supported, though they dont make it easy). Although packing so much data in a single page can be daunting, most users can gain facility with the system within a couple of hours. CapTabulous also allows some powerful sub-models (eg ProFormas) to be exported in excel format for limited-license distribution for free. Note that CapTabulous is NOT just a fancy spreadsheet. It is a comprehensive platform backed by a substantial code base that works behind the scenes, yet requires no add-ins! Pricing has yet to be determined, but will likely start at $5K/seat/yr, a fraction of what companies like Carta charge for what is essentially a glorified database with limited analytics.

The CapTabulous platform consists of two separate but related modelers: the Waterfall Generator, and the Cap Table & ProForma Modeler.

The Waterfall Generator takes as input a Cap Table containing any combination of common stock, preferred stock, options, or debt, each of which can have an accruing compound or simple dividend which pays in cash or stock, and an optional relative seniority. Preferred stock can be non-participating, participating, or participating with a cap. The model is user extensible to allow for alternate accrual formulas, or other customizations, and indeed supports every security the author has ever encountered in his 35 year career in venture capital. Its also fully deterministic, so the results generated are mathematically accurate to the penny. The Waterfall Generator provides 4 separate but related functions:

  The Spot Waterfall shows a variety of payout metrics for a specific exit price on a per-security basis, such as aggregate payout per class, $/sh, MOIC, and % of allocated proceeds, as well as useful thresholds such as the minimum exit price at which each security will receive their full preference, convert to common, or hit an optional cap.

  The Return Curve Generator utilizes the Spot Waterfall to create graphical return curves for a specified range of exit prices per security class, per investor, or per affiliated group of investors. The platform includes an enhancement to Excels somewhat anemic graphic module to allow users to easily zoom in or out of exit price ranges, as well as easily customize the line formats for any plot. Users can also save plots for easy comparison across scenarios. The graphs can also be easily copied and pasted into other programs for reporting and other purposes.

  The Entity Summary allows users to conveniently display a summary of holdings for a particular investor or affiliated investor group, along with useful metrics such as amount invested, aggregate dollars returned, profit, MOIC, and IRR for a specified exit price. The platform also includes an enhanced Solver that can derive the exit price required to achieve a desired outcome (eg, target MOIC or IRR), and Co-Investor Search to display investments made by any entity in other Cap Tables stored within other cap table sheets.

  The Integrator was designed as a vastly improved upgrade to the mathematically dubious methodologies that audit, accounting, and valuation firms have historically used for FMV, 409A, and ASC820 fair market value determination. It uses the mathematical definition of expected value (see https://en.wikipedia.org/wiki/Expected_value) to determine the FMV of a security or basket of securities based upon a probabilistically weighted/unweighted integration of outcomes across a range of likely exit prices for a designated return curve. Prior to the ability to derive accurate return curves, practitioners relied on a combination of conversion thresholds, a modifed black-scholes formula, and questionably relevant public comps and public market volatilities to determine what the author considers to be a notoriously inaccurate assessment of fair market value (even Myron Scholes agrees that practitioners use of his Black-Scholes formula is inappropriate for private company valuations as calculated in the vast majority of 409a reports that the author has seen).

The Cap Table & ProForma Modeler is where users enter the securities, debt, convertible notes, and individual shareholdings for company capitalization tables, and where they can model out ProForma rounds, Preemptive rights, and Scenarios. By necessity, this is a non-deterministic iterative model that relies primarily on Excels iteration mode, but has a backup custom iterator for more complex models. Like the Waterfall Generator, it also consists of 4 main components, and a variety of options:

  The Equity Securities table is where users enter aggregate details of each security in the company cap table, such as each securitys name, price per share, number of shares outstanding, issue date, and (for options) strike price. This is also where you code other terms of the securities, such as the security type (equity, debt, or convertible), seniority, and other terms (eg, participation). Its also where you configure you can figure the terms for an ProForma rounds.

  The Shareholdings table is where users enter the details related to individual holdings of securities. Each line represents the purchase of an individual security class, so holders of multiple types of securities will have multiple entries. Each line contains a security class, Holder name, optional Affiliation, purchase date, and (for equities) the number of shares purchased, or (for notes/convertibles) the dollar amount invested. Once populated the table also includes information metrics such as the %owned, accrued interest, original purchase price, and (if the company is loaded into the waterfall) the proceeds associated with the holding.

  The Convertibles & SAFEs table is where users configure convertible and SAFE notes that were previously added in the Equity Securities table (ie, you never enter new rows here, just fill in rows that were previously added via an entry in the Equity Securities table). The note configuration data inputs are part of a sophisticated abstraction that is flexible enough to represent virtually any flavor of convertible note or SAFE, including multiple recursive, self-referencing SAFEs (where you have to know how one converts to know how the other converts, and vice versa), and notes that depend upon additional option issuances or custom accruals.

  The Cap Sheet Pivot is where users can view a traditional columnar cap table with securities names in the top row, and the shareholdings of each investor or affiliate on successive rows. Its also where you can specify any participants in any preemptive rights offering associated with a new round. You can sort and arrange the cap table using all of the functionality afforded by Excel to a standard pivot table! The Cap Sheet Pivot is configured through the CapTabulous tab in the standard excel ribbon, and can be configured to display either individual holders or affiliates, and display holdings either by nominal shares, as-converted shares, or invested capital.

The Cap Table & ProForma modeler also includes a variety of other global settings, such as the number of significant digits in the share price (which governs share rounding), interest rate parameters for dividends (eg, compound vs simple interest and number of periods per compound rate), ProForma Valuation type (Pre-Money vs Post-Money), and Antidilution enablement. The CapTabulous Excel tab also contains additional useful functions such as a general purpose solver optimized for cap table proforma calculations, a formula auditing tool to restore customized cell formulas, iterations settings, and a preemptive rights configuration tool. It also has the ability to customize built-in functions to allow for non-standard accrual formulas and custom antidilution calculations.

Waterfall Modeler

The Waterfall Modeler is where you generate spot waterfalls and full-spectrum return curves for any cap table, as well as calculate FMVs for FASB ASC 820 and 409a valuations. Return curves can be generated for individual securities, investors, or an aggregate of affiliated investors, and can illustrate a variety of metrics such as $/Sh, Profit, MOIC, or % of distributed proceeds. It supports all known securities the author has encountered in his 35 year career in venture capital. The Waterfall sheet is illustrated below:

Waterfall Cap Table Entry

Use the menu under Load into Waterfall button to load your cap table from your Cap Table sheet into the waterfall. Cap Sheet data populates the columns under the Data Inputs from Cap Table heading. Entries shown in blue reflect numbers that were entered directly in the cap table sheet (vs those that are calculated), but you can overwrite any cell under this heading. The inputs self explanatory, other than the following:

Dividend The Dividend field offers a variety of non-obvious options documented here:

  Numbers < 100% are interpreted as an annual dividend that accrues yearly per the Rate Type (eg, simple,compound) and Periods/Yr options in the cap table sheet.

  Numbers >= 100% are interpreted as a fixed, non-accruing preference multiple.

  Negative numbers are treated like positive ones, except the dividend accrues on the number of as-converted shares instead of the preference.

  Custom Accruals there are 3 ways to utilize a user-customized accrual formula:

o   Enter a formula in the Dividend column to calculate the accrual directly. The result of the formula will be treated as a fixed accrual, without regard to time elapsed (so factor that into your formula.

o   Use a rate type of Custom and redfine the global lambda named CalcAccrual to yield the desired result. All Cap Sheets utilizing the Custom rate type will use this accrual formula).

o   If you use lots of custom accrual formulas, you can create a local-scoped copy of the CalcAccrual lambda in the source Cap Table sheet. Whenever that sheet is loaded into the waterfall, the local lambda from that sheet will preempt the global one. Consult your favorite AI to learn about lambdas and local vs global scoping of Excel Defined Names.

Exit Date for Accruals Calculates values as if exit occurred on this date (plus Years to Exit if it is non-zero). Only relevant if cap table has securities with accruing dividends.

Scenario Modeler

The Scenario Modeling function is perhaps the most powerful part of the CapTabulous modeling tool. Scenarios are used to assess how new securities will affect a company's cap table, or to compare different financing alternatives.

Securities are assigned to a scenario (A-D) using the dropdown box in the Scenario column of their row in the Waterfall Sheet (or in a stored cap table sheet).

  if no scenario is designated for a security (ie, it is blank), or the designated scenario matches the Active Scenario selected in the Scenario: dropdown menu in column L, then that security is treated as though it is part of the current cap table.

  Note that even if a security's assigned scenario is NOT the Active Scenario, its return curve will be plotted as if that security WAS part of the cap table (but all other return curves with a blank scenario will be plotted as the securities in the non-active scenarios didnt exist). For example, if you want to compare the return curves for 2 different prospective securities to be used in a pro-forma (e.g. a participating vs non-participating) you can assign one to scenario A, and another to scenario B and the return curves for each will be plotted as though the other didnt exist. If the active scenario is blank, return curves for all securities with a blank scenario will be plot as though neither the security in scenario A or B is issued. If the active scenario is A, then all return curves will be plot assuming security A is issued (but security B NOT issued since it is not part of that scenario). Note that all securities with blank scenarios are presumed to exist in all scenarios.

  The "" scenario is a special case used to plot exit proceeds for the aggregate holdings of an individual shareholder or group of affiliate shareholders alongside a plot that would not normally include this option (e.g. a plot of securities). To use this, add a row to your Cap Summary table in a Cap Table Sheet, assign it a scenario of "", and put the name of a Holder in the Security column. This is of limited usefulness but it is included for backwards compatibility of earlier CapTabulous versions that didnt have Saved Plots.

Return Curve Plots

Perhaps the most powerful feature of the CapTabulous platform is its ability to visualize Return Curves. Plots can be generated per security, per investor (aggregating all of their held securities) or per group of affiliated investors. Users have easy controls to include/exclude plot lines, save plots, customize the display, and even zoom in and zoom out (a feature Excel itself otherwise lacks). Users can plot aggregate return dollars, $/Sh, % of overall proceeds, and MOIC (for investors/affiliates you can also plot profits).

Plot Options The following options are available to customize your plots:

X-Axis Interval Sets the interval between horizontal axis (ie Exit Price) tick marks. The plot will be centered on the current Exit Price.

Interactive Enables CapTabulouss proprietary extensions to Excel graphs, such as the info popup box and zoom in/out support. Disable if you want direct access to Excels built-in chart functions (eg if you want to manually change the Axes or plot points).

Show Thresholds When plotting by Security, checkmark this option to display a visual indication where the security recovers its full preference, converts to common or (for participating securities) starts participating, hits its cap, or converts.

Use Line Labels Checkmark to insert line labels at the right edge of the plot in lieu of a legend.

Freeze Y Axis Checkmark to freeze the Y-Axis for subsequent plots. Useful for visualizing the difference between scenarios.

PlotLine Customization CapTabulous lets you customize the line formats used to differentiate between different security types. Right-Click the line type shown in the legend and select Format Shape to customize any line types.

Graph Zooming CapTabulous extends Excels Graphics capability to allow you to easily zoom in an out graphs to focus on relevant sections of your return curves.

Zoom In To zoom in on a plot, right click on the X-Axis (Mac users will have to select the plot first). Release the mouse and drag to the left or right to highlight the area you want to zoom. Then click again in the X-Axis to zoom in, or click anywhere in the plot area to cancel.

Zoom Out To zoom out, click anywhere in the plot area. The zoomed-out plot will be centered around the X-Axis value above where you clicked.

Solver Select a target cell youd like to optimize to a particular value by iterating on the Exit Price (the cell must contain a formula),and select Solver from the CapTabulous Ribbon.

Enter the desired target value and click Solve. If the value is not reached within the specified Max Iterations, either increase Max Iterations, or just click Solve again. You can also enter the first Exit Price youd like to try in the Iterator Seed field. For example, if you want to find the exit price that will yield an IRR of 20% for the current Investor Profile, select the cell labeled IRR, click Solver, and enter 20% as the Target Value.

 

Waterfall Modeler Notes & Considerations

Notes

(1)                Blank Dates are interpreted as today, blank Caps are treated as non-participating preferred, blank strike prices are treated as issued equity (vs options), and blank $/Sh are treated as common stock.

(2)                Caps for participating preferred securities are inclusive of any accrued dividends. If you need the cap to be exclusive of the dividend, just increase the cap accordingly. Make sure the accrued dividends are less than the cap, or the results will be inaccurate (eg if you have a cap of 1.5 but accrued dividends of over 1.5x the model wont know how to interpret that).

(3)                You can edit the line styles used for the security types by selecting the line in the Line Legend (not the chart) and right-clicking -> "Format Shape".

(4)                Do not change the names of any tables in any worksheet. This will break a bunch of stuff.

(5)                Do not change the names of table columns, but feel free to add your own columns.

(6)                Excel's calculation process is asynchronous and sometimes doesn't complete before graphs are plotted. If a graph looks incorrect, click RePlot.

(7)                DO NOT insert rows into the Waterfall Sheet Securities Table (change the Cap Table source sheet instead and then reload into the waterfall). You can also change any value in the Data Inputs from Cap Table columns for ad-hoc modeling, but these changes will not be made to the source data sheet.

(8)                Thesholds are not valid for securities that are part of non-active scenarios.

ProForma Modeler

Overview

The CapTabulous ProForma Modeler (the "Modeler") was designed as a prequel product to my CapTabulous Waterfall and Scenario tool, which takes the output of the Modeler and creates a waterfall based on a range of outcomes, as well as return curves and other analytics. The Modeler was designed to address 5 important and previously irksome tasks associated with calculating the cap structures of venture backed C-Corps (and to a lesser extent LLCs):

1)        Cap Table Summary and Shareholder Registry

First and foremost, the Modeler provides a comprehensive framework for documenting the current capital structure of a company, providing a database for storing most of the details of a companys shareholdings, including: Outstanding Convertible Notes/Debt/SAFEs ("Convertibles"); Equities; Individual Holdings; and an easy-to-visualize summary of all issued instruments. The user can provide a host of data including participation, conversion metrics, seniority, dividend, issue date, etc. relevant to each instrument.

2)        Convertible Note Abstraction & Conversion

Perhaps the most powerful and unusual feature of the Modeler is its ability to capture virtually all aspects of an enormous variety of Convertibles. The Modeler supports multiple layered pre-money and post-money SAFEs and other convertible instruments having a broad array of interest rates, conversion discounts, valuation caps, and option pool adjustments. Historically, the most complex aspect of dealing with Convertibles is the fact that the majority of them are documented in a way that makes their conversion non-deterministic; typically, Convertible Notes have provisions whereby the shares into which they convert require the knowledge of how other Convertibles convert, requiring a recursive or iterative solution. This is especially prevalent in SAFE notes which often convert based on either a Discount or Valuation Cap. For example, the share price at which a SAFE converts pursuant to a Valuation Cap requires dividing such Cap by a base capitalization, which can include the Convertible itself, other converting SAFEs, and an option pool expansion (typically expressed as a percentage of the post-conversion capitalization)! The Modeler allows you to enter all the details of any number of Convertibles, and magically calculates the resulting shares into which they should convert.

3)        ProForma Capitalization

The heart of the Modeler is designed to illustrate the ProForma capitalization resulting from a prospective issuance of equity securities of a given investment amount, pre/post-money valuation, and optional pre-round option pool expansion (expressed as a percent of the post-financing capitalization). Users can specify whether the new round triggers the conversion of any outstanding Convertibles, as well as create different scenarios with varying finance terms.

4)        Antidilution Calculations

There are times when a ProForma round might trigger antidilution in existing securities. If this is the case you can have the Modeler automatically calculate any antidilution adjustments and have such adjustments factor into any Convertibles conversions or ProForma calculations. The Modeler supports Broad and Narrow Based Weighted Average antidilution by allowing the user to easily select which existing share classes factor into the standard calculation. The user can also easily customize the formula used by redefining the BBAntidilution formula (scope it to the local sheet).

5)        Interest Accrual

Convertible notes and preferred securities can accrue interest and dividends which can be calcuated as simple or compound, and payable over a variable number of periods per year. Custom interest calculations are also supported by modifying the CalcAccrual lamda function for the local sheet. As investors may purchase the same notes or securities at various closings, interest is calculated on a per-investor basis for the greatest accuracy. For convertible notes, you can specify if interest is included when the shares convert to equity, or only if they redeem for cash. You can also specify whether a note's seniority changes subsequent to its conversion.

6)        Normalization

One feature that most models miss is the fact that share prices are by necessity limited to a fixed number of decimal places (as they are called out in the Company's Articles of Incorporation), and that convertible securities and proformas for which investors generally commit a round number of dollars at some point must convert into an integral number of equity shares. As conversion shares are also factored into things like option pool expansions, rounding errors can add up, especially if you have a large number of participating shareholders. The CapTabulous model takes this all into account and even lets you specify the number of decimal places used for equity share prices. The model allows you to specify dollar-denominated allocations of convertible note principal and proforma round participation, and converts these into integral share numbers upon the closing of a conversion round. For example, if an investor commits $1M to a proforma round, the model will report the exact actual amount of the check the investor needs to write.

7)        Preemptive Rights Calculator

Once youve entered your cap tables shareholders, you can create a conventional cap table for your company using the CapSheet Pivot Table Options button in the CapTabulous ribbon. This creates a standard Excel Pivot Table with some extra useful columns. Once youve created the pivot, select the Preemptive Rights button and youll be able to easily calculate the preemptive rights for each Holder/Affiliate.

Cap Table Modeler Instructions

General

In almost all cases, cells which are meant to take user input are shown in blue text (or have a blue background or border if the input is optional)

ENTER CAP TABLE DATA

Step 1: ENTER YOUR COMPANY'S SECURITIES INTO THE "EQUITY SECURITIES" TABLE

Enter each security name & type (eg Pref A, Pref B, Common, Options...) into the Equity Securities table. Expand the number of rows available by using the Add Row command (multiple rows can be added/deleted by selecting a number of rows before clicking add/delete). By default, all securities start out as non-participating or common/options. You can customize the security type (eg, Note/Debt/ProForma/Participation) later by selecting its row and clicking the "Edit Security" button. The following inputs should be entered directly into the table cells: Scenario, Cap, Rank, $/Sh, Security Name, Shares, Strike Price, and Dividend (for securites designated as ProForma you also enter Investment, Valuation, and Pool%). The Issue Date is calculated based upon the issued dates entered in the Shareholdings table later (use the "Edit Security" button to specify the entry date and customize the security type). Other fields are for documentation purposes and are also used by the CapTabulous Waterfall tool. explanation of the columns in the Equity Security tables are below:

Scenario: Securities can be tied to scenarios A-F which can be activated by the scenario menu (multiple scenarios can be active simultaneously)

Rank: Seniority of the security (lower number is more senior)

Cap: Security type :

-1 is debt (share counts are disregarded other than for cost/preference calcs)

0 is non-participating preferred/common/options (if share price is specified, holders can either redeem for preference or convert) 1 is participating preferred (holders get both their preference AND their conversion shares)

>1 is participating with a cap (eg 2 participates with a 2X cap, so users get their pref + shares but only until they get 2X their investment)

Other Fields are Self Explanatory (and if not you probably should seek other employment).

Do NOT change the names of the built in "Unissued Option Pool" or "ProForma Pool Expansion" security types!

Step 2: CONFIGURE ANY CONVERTIBLE NOTEs/SAFEs

edit the security from the Equity Securities table that is issued pursuant to a Note/SAFE conversion. Choose "Convertible Security" as the security type, and select the base security into which the note will convert (the security will take on the CAP/RANK/ISSUE DATE of the underlying security). All Convertibles are assumed to convert to the same base security. Check "subseries" if the note will have a liquidation preference equal to its own conversion price versus the share price of the base security (ie have a preference equal to the converting principal/interest like a typical SAFE conversion).

Once you have designated a note, it will appear in the Convertibles & SAFEs table at the top of the sheet, where you enter the specifics (this table is hidden if no convertible securities have been specified). Only the following parameters take user input (the rest of the columns are informational. don't try to add/del rows from the note table):

Principal: Note Principal amount

INTEREST PARAMETERS

Issue Date: Date note starts accruing interest (informational only if no interest)

Rate Type: built in types are none, or annual simple or compound interest. Select custom for other types and configure as below.

Rate: Annual Rate. For Custom-type, enter excel formula yielding accrual from "[@[Issue Date]]" to named variable "ConversionDate".

Payments/Yr: for Compound rates, enter the number of compounding periods per year.

When/How Paid: Specify whether accrued interest also converts into shares, or whether it is only paid in the event of a redemption.

CONVERSION PARAMETERS

Discount: Discount (if any) from issue price of base security into which note converts (eg 10% if the note converts at a 10% discount).

Valuation Cap: Calculates max share price at which note will convert, calculated as the Valuation Cap/(Base Securities+Additions).

Base Securities: Specify all securities included in the denominator when converting the Valuation Cap into a price/share at which the note converts. (eg, for a a SAFE with a post-money valuation cap, you would include the SAFE itself in this selection).

Additions: Enter a hard number or a percentage of the Base Securities (exclusive of additions) to add to the aforementioned denominator. ( ie specify 10% to add 10% to the denominator exclusive of the additions, or 10%/(1-10%)= 10.111%. to increase denominator by 10% inclusive of the Additions).

Step 3: CONFIGURE A PRO-FORMA SECURITY, IF NEEDED

A) Any Security in the Equity Securities Table (other than a convertible note/SAFE) can be specified as a Pro-Forma. Simply edit the security and check the ProForma checkbox. Then enter the ProForma Valuation, Investment, and Pool% which will be highlighted in blue. The Pool% is expressed as % of the post-closing capitalization that are reserved for unissued options, and any increase in the pool above the pre-existing Unissued Options will be considered to have happened pre-money. Use the Valuation Type dropdown above the Valuation column to specify if you are quoting a pre-money or post-money value. Usually the ProForma round is also the round that will convert your outstanding Convertibles (not mandatory though). The ProForma assumes that all securities with an issue date that precede the ProForma round issue date are included in the pre/post money valuation. For example, if you want your option pool expansion to occur POST-FINANCING, you should specify 0 as the ProForma Pool% and then add your own extra Option Pool row and give it an issue date AFTER the ProForma round issue date, so the pool expansion would NOT be incorporated into the proforma round price. A proforma round is presumed to close in the user-inputed date marked "Date for Dividend & Interest Accrual/Proforma Closing". This date is relevant if convertible notes which accrue interest convert upon the closing of the proforma.

You can specify as many ProFormas as you like but each ProForma must have a unique issue date. If you have multiple ProFormas specified, the model will assume that for all ProFormas other than the earliest dated one, that the unissued option pool has been exhausted immediately prior to the ProForma date (ie, the ProForma will not net any pre-existing unissued options against the Pool% specified). All ProFormas take into account any option pool expansions that occured in earlier ProFormas into their share price calculation (and the earliest proForma takes into account any pre-existing unissued options). Note that Antidilution is not supported if more than 1 ProForma is specified.

B) CONFIGURE ANTIDILUTION PARAMETERS IF TRIGGERED BY PROFORMA

if the ProForma cacluated share price is lower than any of the previously issued preferred, the AntiDilution options are activated, giving you the option of applying Antidilution to the affected securities. The default Antidilution is the standard weighted average antidilution, but you must specify the securities that are considered to be part of the pre-existing fully diluted shares by specifying them by clicking the cell marked "Base Shares for Antidilution" (If maros are disabled, you have to enter each security manually seperated by a newline (ie Alt-return)). You can implement broadbased or narrowbased antidilution depending on whether you specify whether options and warrants are included (the base shares can be viewed next to the label "Existing Shares for AD Calc". The Antidilution shares are considered to be issued prior to the ProForma round that triggers the antidilution, so they affect the share price of the pro-forma. Specify which securities have antidilution protection by clicking in the cell below the column marked "Has Antidilution Protection" (only visible when antidilution dropdown set to TRUE).

Step 4: ENTER INDIVIDUAL SHAREHOLDINGS

The Shareholdings table is used to show which shareholders hold which securities. Add/Delete rows by selecting a number of rows and using the Add/Delete buttons, or just use Excel's Insert Table Row features (in which case you'll need to fill the appropriate formulas down). The Index and Active Columns are formulas which show the index of the security into the Equity Securities Table and whether that security is part of a currently active scenario. The Date Issued, $/Sh, and Cost columns are derived from the Summary Table but can be overridden (eg if a holder has a different cost because they received a prior discount). For securities that have already been issued to the holder, enter the number of fully diluted shares held, with each security type on its own row. For ProForma investments and Convertibles Held, enter instead the Nominal Investment amount, and the model will figure out the pro forma shares the investor will receive.

Preemptive Rights / Pro Rata Calculator

Starting in column AA of each Cap Table sheet is a modeler that automatically calculates preemptive rights or pro-rata splits for new investments in the company. If a proforma has been defined in the Equity Securities table, simply select that security from the Proforma security popup, and the round size will be picked up from there. Alternatively, just enter the round size into cell marked Round Size (Override).

Scenarios

The Model has the ability to maintain different scenarios, with multiple scenarios active at the same time. Use the Scenarios column in the Equity Securities table to add an equity to a Scenario. Then use the Active Scenario dropdown to select which Scenarios are active (selecting a Scenario will add it if it is not activated, and de-activate it if it is). Equities which are part of an inactive scenarios are treated as though they aren't issued.

As-Of Date

Use the popup menu labeled "View Cap Table As-Of" to view the cap table as-of a specific issuance date. Using this feature you can view the cap table as of the date of a particular round, and disregard securities that were issued after. This is useful for visualizing a cap table before/after a ProForma Round.

Summary Pivot Table

Once you have entered your data and fully configured your model (or before, but it may slow operation) select an option from the Cap Summary Pivot dropdown to create a 'classic' summary cap table with holders (or affiliates) listing in a vertical row, and their shareholdings for each security in each column, with customizable summary columns at the end (see Customizations below).

System Requirements

The Modeler requires a relatively up to date version of excel that supports the "=Lambda" function, Dynamic Arrays, and (for PivotTable extension columns only) the "=MakeArray" function. All of these were introduced in 2021 in the Microsoft 365 version of Excel (formerly Office 365) and by now should be supported by all Excel versions. Although the Modeler makes extensive use of macros (which need to be enabled) to configure your model, once configured it will run without them (you won't be able to add securities or convertibles, but you can change all parameters, and even add new Holders using Excel's "Insert Row" table function. Configured models can also run in the web version of Excel! Note that if macros are disabled you may have to manually enable "Iterative Calculation Mode" (go to Files->Options->Formulas and checkmark Interative Calculation) if you get a circular reference error. Configuring the model assumes a reasonable facility with Excel, although viewing configured models does not.

Limitations

While the Modeler was designed to be extensible and customizable, there are certain limitations imposed by the fact that the model was designed to run using Excel's native capabilities. Some things to keep in mind below:

1) Each security must have a unique name. (eg. You can't have 2 scenarios name Pref A)

2) You can't change the name of the "Unissued Option Pool" or "ProForma Pool Expansion", as certain macros and formulae depend upon them being constant.

3) The Summary Pivot Table must be manually refreshed to reflect changes in the model (right click the pivot table and select "Refresh").
